1 Overview & Quick Start

ElectraGuard accepts .xlsx, .xls, and .csv files. The app auto-detects your column names by matching them against a library of common aliases — so you don't need to rename anything in most cases.

Minimum requirement: At least one numeric column representing electricity consumption (kWh). Everything else is optional but improves detection accuracy.

Two Detection Modes

A

Statistical Detection (always active)

Works with any spreadsheet. Uses Z-Score, IQR, billing ratios, load analysis, and consumption change detection. Needs only a Consumption column.

B

LSTM Deep Learning (activates automatically)

Trains an in-browser neural network when your data includes ≥ 7 time-series columns (e.g. day_1 through day_30). Based on Kocaman & Tümen (2020). Provides a learned theft probability per consumer.

The app looks for the following fields. For each, it tries multiple aliases — use any of the names listed and the app will map it automatically.

Column matching is case-insensitive and ignores special characters. So Consumption (kWh), consumption_kwh, and CONSUMPTION all map to the same field.

3 Time-Series Columns (for LSTM)

To activate the LSTM deep learning engine, your spreadsheet must include at least 7 sequential time-period columns representing daily (or periodic) consumption readings per consumer.

Accepted Naming Patterns

day_1, day_2 …
Daily readings
d_1, d_2 …
Shorthand daily
reading_1, reading_2
Reading index
week_1, week_2
Weekly readings
month_1, month_2
Monthly readings
2025-01-01, 2025-01-02
Date as column header
Minimum 7 time-series columns required for the LSTM to activate. More data points = better temporal pattern learning. 30 daily readings is recommended.

Example Layout

Consumer ID | day_1 | day_2 | day_3 | ... | day_30 | Flag
EG-01001   |  8.5  | 9.2   |  7.8  | ... |  8.1   |  0
EG-01002   |  2.1  | 0.3   |  0.0  | ... |  1.5   |  1    ← theft
EG-01003   | 12.4  | 11.8  | 13.0  | ... | 12.6   |  0

Each row = one consumer. Each day_N column = that consumer's consumption reading on day N. The LSTM learns the temporal pattern across these columns to detect anomalies.

4 Theft Flag / Label Column

The Flag (or Label) column tells the LSTM which consumers are known theft cases. This enables supervised learning for higher accuracy.

Label Values

ValueMeaningDescription
0 Normal / Honest This consumer has no known theft or tampering history. They are a legitimate consumer.
1 Theft / Fraudulent This consumer is a confirmed or suspected electricity thief — identified through physical inspection, meter audit, or prior investigation.
No labels? No problem. If you don't have a Flag column, the app will auto-generate pseudo-labels using statistical heuristics (very low consumption, high variance, zero readings, IQR outliers) to bootstrap the LSTM training.

How Labels Are Used

1

Training Data Split

The LSTM uses your labels to create a supervised classification problem: it learns what normal (0) vs theft (1) consumption patterns look like over time.

2

5-Fold Cross-Validation

Data is split into 5 folds. Each fold takes turns being the validation set. This produces reliable accuracy, precision, recall, and F1 metrics.

3

Theft Probability Output

After training, the LSTM produces a theft probability (0–100%) for each consumer. This is fused with statistical scores into the final hybrid risk score.

6 How Data Is Processed Internally

Understanding the internal pipeline helps you prepare better data. Here's the full flow based on the Kocaman & Tümen (2020) methodology:

1

Data Cleaning

Missing/NaN values are imputed with the column mean. Negative values are set to 0. Empty rows are skipped.

2

Min-Max Normalization

Each consumer's time-series is scaled to [0, 1] using: X_norm = (X - X_min) / (X_max - X_min). This ensures fair comparison across different consumption scales.

3

Sliding Window Sequencing

The normalized series is split into overlapping windows of 7 time-steps. Each window becomes one training sample for the LSTM.

4

LSTM Training

Architecture: LSTM(128) → Dropout(0.2) → Dense(64, relu) → Dense(1, sigmoid). Trained for 30 epochs with Adam optimizer and binary cross-entropy loss.

5

5-Fold Cross-Validation

Model is trained and evaluated 5 times on different data splits. Reports averaged accuracy, precision, recall, and F1 score.

6

Hybrid Risk Scoring

LSTM theft probability (0–40 pts) is combined with statistical flags (0–60 pts) into a final 0–100 risk score per consumer.

7 Best Practices & Tips

✅ Do's

  • Use consistent column names across your header row — no merged cells or multi-row headers.
  • Include as many columns as possible for maximum detection accuracy.
  • For LSTM: provide ≥ 7 daily reading columns. 30 days is ideal.
  • If you have historical theft records, add a Flag column (0/1) for supervised training.
  • Keep data on the first sheet of your Excel file.
  • Use numeric values for consumption, billing, and load columns (no text like "N/A").
  • Label at least 10% of your data as theft (Flag=1) for balanced LSTM training.

❌ Don'ts

  • Don't use merged cells — the parser won't handle them correctly.
  • Don't put data starting from row 3+ with blank rows above — data must start at row 1 (header) + row 2 (first data row).
  • Don't mix units in the same column (e.g., some rows in kWh, others in MWh).
  • Don't use commas in numeric values (e.g., use 1500 not 1,500 in the raw cell).
  • Don't leave the Flag column with values other than 0 or 1.
