1 Overview & Quick Start

ElectraGuard accepts .xlsx, .xls, and .csv files. The app auto-detects your column names by matching them against common aliases — so you don't need to rename anything in most cases.

Minimum requirement: At least one numeric column representing electricity consumption (kWh). Everything else is optional but improves detection accuracy.

Two Detection Modes

A
Statistical Detection (always active)

Works with any spreadsheet. Uses Z-Score, IQR, billing ratios, load analysis, and consumption change detection. Needs only a Consumption column.

B
LSTM Deep Learning (activates automatically)

Trains an in-browser neural network when your data includes ≥ 7 time-series columns (e.g. day_1 through day_30). Based on Kocaman & Tümen (2020).

3 Time-Series Columns (for LSTM)

To activate the LSTM deep learning engine, include ≥ 7 sequential time-period columns representing daily consumption readings per consumer.

Accepted Naming Patterns

Shorthand daily

Reading index
week_1, week_2
Weekly readings

Monthly readings

Minimum 7 time-series columns required for LSTM. More = better. 30 daily readings is recommended.

Example Layout

Consumer ID | day_1 | day_2 | day_3 | ... | day_30 | Flag
EG-01001   |  8.5  | 9.2   |  7.8  | ... |  8.1   |  0
EG-01002   |  2.1  | 0.3   |  0.0  | ... |  1.5   |  1    ← theft
EG-01003   | 12.4  | 11.8  | 13.0  | ... | 12.6   |  0

Each row = one consumer. Each day_N column = that consumer's reading on day N. The LSTM learns the temporal pattern across these columns.

4 Theft Flag / Label Column

The Flag column tells the LSTM which consumers are known theft cases. This enables supervised learning.

Label Values

ValueMeaningDescription
0 Normal / Honest No known theft or tampering. Legitimate consumer.
1 Theft / Fraudulent Confirmed or suspected thief — identified through inspection, audit, or investigation.
No labels? No problem. If you don't have a Flag column, the app will auto-generate pseudo-labels using statistical heuristics to bootstrap the LSTM training.

How Labels Are Used

1
Training Data Split

The LSTM learns what normal (0) vs theft (1) consumption patterns look like over time.

2
5-Fold Cross-Validation

Data is split into 5 folds, producing reliable accuracy, precision, recall, and F1 metrics.

3
Theft Probability Output

After training, the LSTM produces a theft probability (0–100%) per consumer, fused into the hybrid risk score.

6 How Data Is Processed Internally

The pipeline based on Kocaman & Tümen (2020):

1
Data Cleaning

Missing/NaN values → column mean. Negatives → 0.

2
Min-Max Normalization

X_norm = (X - X_min) / (X_max - X_min) — scaled to [0, 1] per consumer.

3
Sliding Window Sequencing

7-step overlapping windows as LSTM input samples.

4
LSTM Training

LSTM(128) → Dropout(0.2) → Dense(64,relu) → Dense(1,sigmoid) · Adam · 30 epochs.

5
5-Fold Cross-Validation

Averaged accuracy, precision, recall, F1 across 5 splits.

6
Hybrid Risk Scoring

LSTM probability (0–40 pts) + Statistical flags (0–60 pts) = 0–100 risk score.

7 Best Practices & Tips

✅ Do's

  • Use consistent column names — no merged cells or multi-row headers.
  • Include as many columns as possible for maximum detection accuracy.
  • For LSTM: provide ≥ 7 daily reading columns. 30 days is ideal.
  • If you have historical theft records, add a Flag column (0/1).
  • Keep data on the first sheet of your Excel file.
  • Use numeric values for consumption, billing, and load columns.
  • Label at least 10% of your data as theft (Flag=1) for balanced training.

❌ Don'ts

  • Don't use merged cells — the parser won't handle them.
  • Don't put data starting from row 3+ with blank rows above.
  • Don't mix units in the same column (e.g., kWh and MWh).
  • Don't use commas in numeric values (use 1500 not 1,500).
  • Don't use Flag values other than 0 or 1.
